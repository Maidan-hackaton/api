Введення

Проект «Відкритий бюджет» надає вимогу розробникам використовувати власну базу даних задля можливості їївикористання при розробці власних додатків. Окрім того, ми самі використовує мо цю модель доступу до зібраних та стандартизрованих нами даних. Інтерфейс доступу використовує модель Representational state transfer (REST). Подробиці та деталі доступу до окремих ресурсів ви знайдете нижче.

Регіони

Перелік суб'єктів бюджетування за територіальним поділом. Суб'єкти ідентифікуються за наступною логікою: 0 — Україна, 1-24 — області.

Адреса ресурсу

GET https://api.open-budget.org/v1/areas.json

Параметри запиту

id опціонально Унікальний ідентифікатор регіону. Допускається перечада декількох ідентифікаторів (див. приклад).
search_query опціонально Пошук входження значення параметру у назву регіону

Приклад запиту

https://api.open-budget.org/v1/areas.json?id=0,1,2,3 (Натисніть для того щоб подивитись результат запиту)

Документи

Законодавчі акти в котрих закріплено поділення надходжень та витрат державного та місцевих бюджетів

Адреса ресурсу

GET https://api.open-budget.org/v1/documents.json

Параметри запиту

area_id опціонально Вказує на регіон, по котрому потрібно зробити виборку. 0 — Україна, 1-24 — області.
Докладніше про регіони дивіться тут. Допускається перечада декількох ідентифікаторів.
id опціонально Унікальний ідентифікатор документу. Допускається перечада декількох ідентифікаторів.
year опціонально Рік набуття чинності документом. Допускається перечада декількох ідентифікаторів.
search_query опціонально Пошук входження значення параметру у назву документу

Приклад запиту

https://api.open-budget.org/v1/documents.json?area_id=1&year=2014 (Натисніть для того щоб подивитись результат запиту)

Витрати

Статті витрат державного та обласних бюджетів, зґруповані за регіонами та роками

Адреса ресурсу

GET https://api.open-budget.org/v1/expenses.json

Параметри запиту

area_id опціонально Вказує на ідентифікатор регіону, по котрому потрібно зробити виборку. 0 — Україна, 1-24 — області.
Докладніше про регіони дивіться тут. Допускається перечада декількох ідентифікаторів.
year опціонально Рік, на котрий запланована певна стаття видатків. Допускається перечада декількох ідентифікаторів.
document_id опціонально Ідентифікатор документу, в котрому задекларована певна стаття видатків.
Докладніше про документи дивіться тут. Допускається перечада декількох ідентифікаторів.
code опціонально Код статті видатків за тимчасовою класифікацією. Допускається перечада декількох ідентифікаторів.
id опціонально Унікальний ідентифікатор статті видатків. Будується з року, на котрій запланована стаття видатків, ідентифікатору регіону та коду статті за тимчасовою класифікацією. Наприклад: стаття з кодом ТКВ № 091206, запланована на 2014 рік у обласному бюджеті Вінницької області буде мати унікальний ідетнифікатор 2014-1-91206
search опціонально Пошук входження значення параметру у назву статті витрат

Приклад запиту

https://api.open-budget.org/v1/expenses.json?area_id=1&year=2014&search=Державне+управління (Натисніть для того щоб подивитись результат запиту)
